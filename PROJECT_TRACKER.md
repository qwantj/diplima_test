# Детальный трекер проекта: DDoS Detection System

Этот документ представляет собой максимально подробный чек-лист всего программного комплекса. Он позволяет детально проанализировать уже реализованный функционал (с галочками `[x]`) и увидеть список идей и задач, которые можно реализовать в будущем (пустые чекбоксы `[ ]`), чтобы довести проект до уровня энтерпрайз-продукта и блестяще защитить диплом.

---

## 1. Сбор и анализ сетевого трафика (Network & Core)

### Реализовано ✅
- [x] **Захват трафика:** Использование `Npcap` и высокоуровневого C++ API `PcapPlusPlus`.
- [x] **Режим Replay:** Чтение и анализ `PCAP/PCAPNG` дампов с сохранением оригинальных временных меток (`timestamps`).
- [x] **Производительность:** Lock-Free очереди (`moodycamel::ConcurrentQueue`) для связи потока захвата и потока инференса.
- [x] **Защита от OOM (Load Shedding):** Отбрасывание пакетов при переполнении очереди (лимит 500,000) с подсчетом `drop_rate`.
- [x] **Маркировка трафика (PcapDumper):** Автоматическое сохранение проанализированного трафика в размеченные дампы `benign.pcap` и `attack.pcap` (потоковая запись без блокировок и утечек RAM).
- [x] **BPF-фильтрация:** Механизм динамического исключения IP-адресов из захвата на уровне драйвера Npcap (отсечение мусора).
- [x] **Извлечение признаков (Feature Extraction):** Агрегация трафика в 2-секундные окна и извлечение 8 метрик CIC-DDoS2019.
- [x] **Гибридная эвристика:** Жесткая пометка трафика как "Атака" при превышении физического лимита интерфейса (например, >20,000 PPS), даже если ML-модель не уверена.
- [x] **Per-Flow Tracking:** Переход от единого 2-секундного окна на весь интерфейс к отслеживанию каждого отдельного соединения (хэш-таблица 5-Tuple: Src/Dst IP, Src/Dst Port, Protocol). Позволит выявлять "тихие" L7 атаки типа Slowloris.
- [x] **Расширенные признаки (Features):** Добавить извлечение TCP-флагов (SYN, ACK, FIN, RST, PSH, URG), энтропии Payload-а и TCP Window Size для более умных ML-моделей.
- [x] **Обучение финальных моделей:** Создан отдельный Python-скрипт `kaggle_pipeline.py` для генерации синтетического датасета, обучения моделей (Random Forest, MLP) с расширенным набором из 16 признаков и экспорта их в релизные `onnx`.

## 3. База данных и Хранение (Database & IPC)

### Реализовано ✅
- [x] **СУБД:** Подключение к `PostgreSQL` (через Qt SQL).
- [x] **Асинхронный Batching:** Сброс данных в БД пакетами каждые 5 секунд в отдельном фоновом потоке, чтобы не блокировать интерфейс и анализ.
- [x] **Схема БД:** Хранение истории сессий (`sessions`), детальной статистики по окнам (`events` и `stats_snapshots`) и инцидентов (`security_events`).
- [x] **State Machine Инцидентов:** Логика автоматического объединения множества атакующих окон в единый инцидент с расчетом длительности и главного атакующего IP.
- [x] **Локальный кольцевой буфер (`FileBuffer`):** Защита от потери данных. Если монитор не подключен, данные копятся локально и пересылаются при подключении.
- [x] **TCP-Протокол:** Версионирование JSON-сообщений (`"version": "2.2"`) по локальному сокету 50050.

## 4. Визуализация и Управление (GUI Monitor)

### Реализовано ✅
- [x] **Дизайн-система:** Профессиональная антрацитовая тема оформления (System/Dark/Light) на базе Yandex Monium.
- [x] **Панель Overview:** Статус подключения, PPS, Drop Rate, Текущая модель, Вероятность, BPF-чекбокс.
- [x] **Area Chart (Трафик):** Интерактивный (Zoom/Pan) график трафика по протоколам (TCP/UDP/ICMP) с полупрозрачной заливкой зоны уверенности атаки.
- [x] **Аналитика (Deep Analytics):** 
    - Матрица здоровья (SLO Alert Grid).
    - Топологическая карта сети (Network Topology).
    - Таблицы Top Sources и Top Targets (активные IP).
    - Кольцевая диаграмма (Pie Chart) топ портов.
    - Тепловая карта активности портов (Heatmap).
    - График пропускной способности (Bandwidth Mbps).
    - Гистограмма распределения размеров пакетов (Bar Chart).
- [x] **Системный мониторинг:** Сбор метрик ОС Windows (CPU, RAM) и их визуализация на кольцевых графиках (Donuts).
- [x] **Журналы и Инциденты:** 
    - Live Event Log с мягкой цветовой кодировкой (зеленый/красный).
    - Вкладка инцидентов с таймлайном угроз (Uptime Bar на 24 часа) и экспортом в CSV.
- [x] **SysTray:** Интеграция в системный трей с динамической иконкой (индикация атаки) и контекстным меню.
