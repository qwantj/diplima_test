## ✅ Исправил Я (ИИ-ассистент)

### 7. `result.PPS` → `result.pps` — листинг 9 (раздел 3.5)
- **Что было:** `if (result.PPS < NOISE_THRESHOLD_PPS)`
- **Что стало:** `if (result.pps < NOISE_THRESHOLD_PPS)`
- **Почему:** в C++ коде (`DetectionEngine.cpp`) поле структуры `DetectionResult` называется `pps` (нижний регистр). `PPS` — только имя колонки PostgreSQL

### 8. `e.result.PPS` → `e.result.pps` — листинг 10 (раздел 3.6)
- **Что было:** `e.result.label, e.result.confidence, e.result.PPS,`
- **Что стало:** `e.result.label, e.result.confidence, e.result.pps,`
- **Почему:** та же причина — строка-имя колонки `"PPS"` в SQL остаётся верной, поле структуры C++ — нижний регистр

### 9. Удалён рабочий чеклист из тела документа
- **Что было:** после рис. Е.6 в конце файла находился рабочий TODO-чеклист (`- [x] Раздел 1.1...`, `- [ ] Добавить скриншоты...`) — 24 строки в теле итогового документа
