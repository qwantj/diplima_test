## ОПРЕДЕЛЕНИЯ (≈ 0.5 стр.)

В настоящем проекте применяются следующие термины:

1. **DoS-атака** (Denial of Service) — атака, целью которой является нарушение нормального функционирования сетевого сервиса.
2. **DDoS-атака** (Distributed Denial of Service) — распределённая DoS-атака, осуществляемая одновременно с множества источников.
3. **Пакет** — единица данных сетевого протокола, содержащая заголовок и полезную нагрузку.
4. **Поток (flow)** — совокупность пакетов, имеющих одинаковые значения 5-tuple: (src_ip, dst_ip, src_port, dst_port, protocol).
5. **Признак трафика (feature)** — числовая характеристика пакета или потока, используемая для классификации.
6. **Классификатор** — алгоритм машинного обучения, относящий объект к одному из заданных классов.
7. **Межсетевой экран (firewall)** — программное или аппаратное средство фильтрации сетевого трафика по заданным правилам.
8. **Правило блокировки** — набор условий (IP-адрес, порт, протокол), при выполнении которых трафик отбрасывается.
9. **SRS (Software Requirements Specification)** — спецификация требований к программному обеспечению; документ, фиксирующий функциональные и нефункциональные требования к системе до начала её проектирования (стандарт IEEE 830).
10. **Функциональное требование (FR)** — требование, описывающее конкретное действие или поведение системы (что система должна делать).
11. **Нефункциональное требование (NFR)** — требование, описывающее характеристику работы системы: производительность, надёжность, безопасность, совместимость.
12. **Критерий приёмки (acceptance criterion)** — измеримое условие, при выполнении которого требование считается реализованным корректно.

**Актуальность.** В современном мире кибератаки типа «отказ в обслуживании» (DoS/DDoS) являются одной из наиболее распространённых угроз информационной безопасности. По данным [источник], ежегодный ущерб от DDoS-атак составляет [N] млрд долларов. Атаки направлены против серверов, сетевой инфраструктуры и критически важных систем: банков, государственных служб, телекоммуникационных провайдеров. Традиционные средства защиты (правила межсетевых экранов, системы IDS/IPS) не успевают адаптироваться к новым видам атак, что обусловливает необходимость применения интеллектуальных методов обнаружения.

**Цель работы:** разработка программного комплекса, обеспечивающего автоматическое обнаружение DoS/DDoS-атак в сетевом трафике в режиме реального времени и блокировку источников атак с помощью правил межсетевого экрана.

4. Разработать алгоритм сбора и нормализации признаков сетевого трафика.
5. Реализовать модуль классификации трафика на основе алгоритма XGBoost.
6. Реализовать подсистему управления правилами межсетевого экрана (`iptables`/`nftables`).
7. Разработать графический интерфейс пользователя для мониторинга в реальном времени.

**1.4.2. Алгоритм градиентного бустинга (XGBoost)**

Алгоритм XGBoost строит ансамбль решающих деревьев. Целевая функция минимизируется итерационно:

```
L(φ) = Σ_i l(ŷ_i, y_i) + Σ_k Ω(f_k)
```

где:
- `l(ŷ_i, y_i)` — функция потерь (логистическая для бинарной классификации)
- `Ω(f_k) = γT + ½λ‖w‖²` — регуляризация (T — количество листьев, w — веса листьев)

На каждой итерации t добавляется новое дерево f_t, минимизирующее:

```
L^(t) ≈ Σ_i [g_i f_t(x_i) + ½ h_i f_t²(x_i)] + Ω(f_t)
```

где g_i = ∂l/∂ŷ — градиент, h_i = ∂²l/∂ŷ² — гессиан функции потерь.

**1.4.3. Метрики оценки качества классификатора**

- Точность (Precision): P = TP / (TP + FP)
- Полнота (Recall): R = TP / (TP + FN)
- F1-мера: F1 = 2·P·R / (P + R)
- ROC AUC: площадь под кривой ROC

**Таблица 4. Матрица ошибок классификатора**

| | Предсказано: атака | Предсказано: норма |
|---|---|---|
| **Реально: атака** | TP (верно определена атака) | FN (пропуск атаки) |
| **Реально: норма** | FP (ложная тревога) | TN (верно определена норма) |

**1.4.4. Нормализация признаков**

Для приведения признаков к единому масштабу применяется стандартизация (z-score):

```
x'_i = (x_i - μ) / σ
```

где μ — среднее значение признака, σ — стандартное отклонение.

**Вывод по разделу:** выбранный алгоритм XGBoost обеспечивает высокую точность классификации при наличии числовых признаков и устойчив к выбросам. Выделенные 8 признаков позволяют эффективно разделять нормальный трафик и атаки всех рассмотренных типов.

### 2.4. Разработка схемы алгоритмов (≈ 5 стр.)

*(По образцу ВКР 3 с блок-схемами, и ВКР 4 с алгоритмами обработки)*

**2.4.1. Алгоритм главного цикла обработки пакетов**

*(Рис. N. Блок-схема алгоритма главного цикла)*

Алгоритм работы подсистемы `ddos_collector`:

1. Инициализация libpcap, открытие интерфейса
2. Загрузка модели XGBoost из файла
3. Загрузка параметров нормализации (μ, σ для каждого признака)
4. Открытие базы данных PostgreSQL
5. Запуск цикла захвата пакетов:
   - Получить пакет от libpcap
   - Вызвать `FeatureExtractor::extract()` → вектор признаков
   - Вызвать `FeatureScaler::scale()` → нормализованный вектор
   - Вызвать `DDosClassifier::classify()` → метка + вероятность
   - Если метка = «атака»: `DatabaseManager::saveEvent()`, `FirewallManager::blockIP()`
   - Иначе: `DatabaseManager::saveEvent()` (с меткой «норма»)
6. При получении сигнала останова — завершить захват, закрыть ресурсы

**2.4.2. Алгоритм извлечения признаков**

*(Рис. N. Блок-схема алгоритма извлечения признаков)*

1. Получить сырой пакет (raw bytes)
2. Определить тип: Ethernet → IP → TCP/UDP/ICMP
3. Извлечь из IP-заголовка: src_ip, dst_ip, ttl, proto, ip_flags
4. Если TCP: извлечь tcp_flags, src_port, dst_port, pkt_len
5. Если UDP: извлечь src_port, dst_port, pkt_len; tcp_flags = 0
6. Если ICMP: тип, код; src_port = dst_port = 0
7. Обновить скользящую статистику по src_ip: счётчик пакетов, байт, время последнего пакета
8. Вычислить pkt_rate = packets_in_window / window_duration
9. Вычислить byte_rate = bytes_in_window / window_duration
10. Вычислить syn_ratio = syn_count / total_count (только для TCP)
11. Вернуть структуру `PacketFeatures`

**2.4.3. Алгоритм блокировки источника атаки**

*(Рис. N. Блок-схема алгоритма блокировки)*

1. Получить IP-адрес источника атаки и вероятность
2. Проверить: не заблокирован ли уже этот IP (запрос к таблице `blocked_ips`)
3. Если заблокирован — обновить счётчик пакетов, вернуться
4. Если не заблокирован:
   - Выполнить: `iptables -I INPUT -s <IP> -j DROP`
   - Записать в таблицу `blocked_ips`
   - Запустить таймер автоматической разблокировки (по умолчанию 300 с)
5. По истечении таймера выполнить: `iptables -D INPUT -s <IP> -j DROP`

// Листинг 1. Инициализация захвата пакетов (TrafficMonitor::start)
bool TrafficMonitor::start() {
    char errbuf[PCAP_ERRBUF_SIZE];
    handle_ = pcap_open_live(
        interface_.c_str(),
        BUFSIZ,       // размер буфера захвата
        1,            // promiscuous mode
        1000,         // timeout (мс)
        errbuf
    );
    if (!handle_) {
        std::cerr << "pcap_open_live failed: " << errbuf << std::endl;
        return false;
    }
    // Установить BPF-фильтр для захвата только IP-трафика
    struct bpf_program fp;
    pcap_compile(handle_, &fp, "ip", 0, PCAP_NETMASK_UNKNOWN);
    pcap_setfilter(handle_, &fp);
    
    running_ = true;
    capture_thread_ = std::thread([this]() {
        pcap_loop(handle_, -1, packetHandler, 
                  reinterpret_cast<u_char*>(this));
    });
    return true;
}

    PacketFeatures f{};
    const auto* ip = reinterpret_cast<const struct ip*>(pkt.data + 14);
    
    f.pkt_len  = ntohs(ip->ip_len);

    PacketFeatures scaled = raw;

// Листинг 4. Классификация (DDosClassifier::classify)
ClassificationResult DDosClassifier::classify(const PacketFeatures& f) {
    float data[8] = {
        f.pkt_len, f.ip_flags, f.tcp_flags,
        f.pkt_rate, f.byte_rate, f.syn_ratio, f.ttl, f.proto
    };
    DMatrixHandle dmat;
    XGDMatrixCreateFromMat(data, 1, 8, NAN, &dmat);
    
    bst_ulong out_len;
    const float* out_result;
    XGBoosterPredict(booster_, dmat, 0, 0, 0, &out_len, &out_result);
    
    ClassificationResult result;
    result.probability = out_result[0];
    result.label = (result.probability >= threshold_) ? 1 : 0;
    
    XGDMatrixFree(dmat);
    return result;
}

bool FirewallManager::blockIP(const std::string& ip, int duration_s) {
    std::string cmd = "iptables -I INPUT -s " + ip + " -j DROP";
    if (!executeCommand(cmd)) return false;
    
    // Запись в БД и автоматическая разблокировка
    db_->saveBlockedIP(ip, duration_s);
    
    // Запустить поток разблокировки
    std::thread([this, ip, duration_s]() {
        std::this_thread::sleep_for(std::chrono::seconds(duration_s));
        unblockIP(ip);
    }).detach();
    
    return true;
}
