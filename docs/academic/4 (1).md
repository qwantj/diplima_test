ВВЕДЕНИЕ

Испокон веков человечество волновала возможность запечатления и сохранения окружающей его действительности. Начиная с созданных первобытных времен и заканчивая современностью, основным методом достижения этой задачи является представление получаемой нами зрительной информации в виде изображения в той или иной форме (например, рисунки, картины, фотографии).

С приходом информационной эпохи появилась возможность сохранения изображений в цифровом виде и их отображение в визуальной форме.

Цифровые изображения бывают двух видов: растровые и векторные [1].

Растровые изображения представляют собой прямоугольный массив пикселей (наименьших элементов), хранящих в себе данные параметров определенного цветового пространства (RGB, CMYK, XYZ, YCbCr и т.п.).

Векторные, в свою очередь, основаны на представлении изображения в виде элементарных геометрических объектов и их описании в виде векторов.

Также появляется понятие цифровой обработки изображений – обработка цифровых изображений с использованием различных алгоритмов с помощью цифровых вычислительных машин. Она включает в себя различные операции: сглаживание, яркостные преобразования, восстановление, реконструкцию, фильтрацию и др. [2].

С развитием технологий возникает необходимость повышения качества изображений для улучшения их зрительного восприятия. Это может быть достигнуто с помощью одного из методов цифровой обработки – масштабирования, результатом которой является увеличение разрешения с сохранением пропорций [3]. Оно осуществляется применением различных математических алгоритмов и способов обработки цифровых сигналов.

По результатам преддипломной практики необходимо разработать приложение для улучшения качества изображений с помощью уже существующих методов масштабирования.

На основе результатов реализации данных методов следует провести их сравнительный анализ.

Итогом работы приложения должно быть получение масштабированного изображения.

2. МАСШТАБИРОВАНИЕ ИЗОБРАЖЕНИЙ

2.1. Структура файла BMP

BMP (от англ. Bitmap Picture) – формат файла растрового изображения, который был разработан компанией Microsoft в 1986 году [5].

В данном формате пиксели хранятся в формате RGBQUAD (целочисленные, могут содержать значения от 0 до 255) [7]:

* Red – яркость красного канала;
* Green – яркость зеленого канала;
* Blue – яркость голубого канала;
* Alpha – альфа-канал (степень прозрачности);

Альфа-канал является опциональным (в зависимости от разрядности изображения). Если он отсутствует, то такой формат называется RGBTRIPLE [4].

Общая структура BMP файла [6] представлена в таблице 1 (структуры представлены в порядке их размещения в файле).

| BITMAPINFO | от 12 до 124 (в зависимости от версии) | Содержит подробную информацию об изображении (формат BMP файла, размерности изображения и др.) |
| Маски | переменный размер | Хранит маски для извлечения каналов в BMP файле |
| Таблица цветов | переменный размер | Нужна для определения цветов по их индексам |
| Массив пикселей | переменный размер | Содержит в себе массив пикселей изображения |

**Маски**

Маски используются для извлечения каналов в BMP файлах с разрядностью равной 16 или 32.

Использование маски зависит от значения поля biCompression (размер указан для разрядностей 16 и 32 бит соответственно) [13]:

* biCompression = 3 – маска формата RGBTRIPLE (размер 12 байт или 24 байт);
* biCompression = 6 – маска формата RGBQUAD (размер 16 байт или 32 байт);

Также маски обязательно присутствуют в BITMAPINFOHEADER версий 4 и 5 и хранится в соответствующих полях (при этом отдельной структуры для хранения масок не создается).

Обязательно создается в BMP файлах разрядностью от 1 до 8 (в остальных случаях не используется).

Если поле biClrUsed = 0, то размерность таблицы цветов равна 2n, где n – разрядность файла.

Если существует таблица цветов, то в массив пикселей вместо значений RGBQUAD будет содержать индексы из таблицы цветов.

2.2. Методы масштабирования изображений

2.2.1. Метод ближайшего соседа

Метод ближайшего соседа (от англ. Nearest Neighbor) также называют интерполяцией нулевого порядка. Его суть заключается в том, что значения интерполируемых пикселей делают равными значениям пикселей, к которым они примыкают [2].

Для двумерного изображения алгоритм реализации данного метода, будет иметь вид (1):

|  |  |
| --- | --- |
| ![](data:image/x-wmf;base64...) | (1) |

, где ![](data:image/x-wmf;base64...) – степень увеличения по оси X; ![](data:image/x-wmf;base64...) – степень увеличения по оси Y; ![](data:image/x-wmf;base64...) – исходное значение ширины; ![](data:image/x-wmf;base64...) – исходное значение высоты; ![](data:image/x-wmf;base64...) – новое значение ширины; ![](data:image/x-wmf;base64...) – новое значение высоты; ![](data:image/x-wmf;base64...) и ![](data:image/x-wmf;base64...) – масштабные коэффициенты; ![](data:image/x-wmf;base64...) – исходная матрица пикселей; ![](data:image/x-wmf;base64...) – масштабированная матрица пикселей.

В результате, метод ближайшего соседа является самым быстродейственным, но в то же время самым низким по качеству итогового изображения.

#экстраполирование краев матрицы на n пикселей

def matrix\_extrapolation(Matrix, H\_old, W\_old, n):

#количество элементов в пикселе

k = len(Matrix[0][0]);

#экстраполируем исходную матрицу

extraMatrix = new\_empty\_matrix(H\_old + 2 \* n, W\_old + 2 \* n, k);

#заполняем середину значениями BMPMatrix

for i in range(H\_old):

for j in range(W\_old):

extraMatrix[i + n][j + n] = Matrix[i][j];

#копируем последовательно краевые пиксели

for m in range(n):

#экстраполируем (копируем) столбцы и строки (кроме углов)

for i in range(H\_old + 2 \* m):

extraMatrix[i + n - m][n - m - 1] = extraMatrix[i + n - m][n - m];

extraMatrix[i + n - m][W\_old + 2 \* n - (n - m)] = extraMatrix[i + n - m][W\_old + 2 \* n - (n - m + 1)];

for j in range(W\_old + 2 \* m):

extraMatrix[n - m - 1][j + n - m] = extraMatrix[n - m][j + n - m];

extraMatrix[H\_old + 2 \* n - (n - m)][j + n - m] = extraMatrix[H\_old + 2 \* n - (n - m + 1)][j + n - m];

#экстраполируем угловые значения (равны среднему значению соседних пикселей)

for i in range(k):

extraMatrix[n - m - 1][n - m - 1][i] = (extraMatrix[n - m][n - m - 1][i] + extraMatrix[n - m - 1][n - m][i]) // 2;

extraMatrix[H\_old + 2 \* n - (n - m)][n - m - 1][i] = (extraMatrix[H\_old + 2 \* n - (n - m + 1)][n - m - 1][i] + extraMatrix[H\_old + 2 \* n - (n - m)][n - m][i]) // 2;

extraMatrix[n - m - 1][W\_old + 2 \* n - (n - m)][i] = (extraMatrix[n - m][W\_old + 2 \* n - (n - m)][i] + extraMatrix[n - m - 1][W\_old + 2 \* n - (n - m + 1)][i]) // 2;

extraMatrix[H\_old + 2 \* n - (n - m)][W\_old + 2 \* n - (n - m)][i] = (extraMatrix[H\_old + 2 \* n - (n - m + 1)][W\_old + 2 \* n - (n - m)][i] + extraMatrix[H\_old + 2 \* n - (n - m)][W\_old + 2 \* n - (n - m + 1)][i]) // 2;

return extraMatrix;

#вспомогательная функция с общей формулой билинейной интерполяции:

#A - f(0, 0);

#B - f(0, 1);

#C - f(1, 0);

#D - f(1, 1);

#x - текущее значение по oX;

#y - текущее значение по oY.

def BLI\_function(A, B, C, D, x, y):

z = int(A \* (1 - x) \* (1 - y) + B \* (1 - x) \* y + C \* x \* (1 - y) + D \* x \* y);

return z;

#Directional Cubic Convolution Interpolation (DCCI) - Направленная Интерполяция Кубической Свертки

#CCI фильтр:

#A, B, C, D - точки из выборки.

def CCI(A, B, C, D):

out = (-1 \* A + 9 \* B + 9 \* C - 1 \* D) // 16;

return out;

#сила краев вверх-вправо (45 град):

#Matrix - рассматриваемый сэмпл;

else:

#Matrix - рассматриваемый сэмпл;

for i in range(0, 2):

p = p1;

else:

w1 = 1 / (1 + G1\*\*k);

#Интерполяция с помощью двухмерного преобразования Фурье

#прямое двумерное дискретное преобразование Фурье:

#p - необязательный параметр (если он есть, то рассматриваем матрицу массивов).

def ft2(u, v, f, M, N, p = -1):

F = 0;

if(p >= 0):

for x in range(M):

for y in range(N):

F += f[x][y][p] \* math.e\*\*(-1j \* 2 \* math.pi \* ((u \* x) / M + (v \* y) / N));

else:

for x in range(M):

for y in range(N):

F += f[x][y] \* math.e\*\*(-1j \* 2 \* math.pi \* ((u \* x) / M + (v \* y) / N));

F \*= 1/(M \* N);

return F;

#обратное двумерное дискретное преобразование Фурье:

#p - необязательный параметр (если он есть, то рассматриваем матрицу массивов).

def ift2(x, y, F, M, N, p = -1):

f = 0;

if(p >= 0):

for u in range(M):

for v in range(N):

f += F[u][v][p] \* math.e\*\*(1j \* 2 \* math.pi \* ((u \* x) / M + (v \* y) / N));

else:

for u in range(M):

for v in range(N):

f += F[u][v] \* math.e\*\*(1j \* 2 \* math.pi \* ((u \* x) / M + (v \* y) / N));

if(abs(f) < 0):

f = 0;

elif(abs(f) > 255):

f = 255;

return int(abs(f));

#среднеквадратичная яркость пикселей:

#p - необязательный параметр (если он есть, то рассматриваем матрицу массивов).

def rms(f, M, N, p = -1):

out = 0;

if(p >= 0):

for x in range(M):

for y in range(N):

out += f[x][y][p];

else:

for x in range(M):

for y in range(N):

out += f[x][y];

out \*= 1 / (M \* N);

return out;

#input - информация о bmp-файле;

#n - степень увеличения;

#sample\_x - размера сэмпла по oX, который интерполируется преобразованиями Фурье;

#количество информации в пикселе (RGB - 3 или с альфа-каналом - 4)

else:

outMatrix = BMPMatrix;

#Интерполяция с помощью фильтра Ланцоша

#ядро Ланцоша по дискретным отсчетам (для одномерного случая):

#x - значение по oX;

#N - дискретный размер ядра Ланцоша.

def L(x, a = 2):

if (x == 0):

out = 1;

#x - значение по oX;
