# Руководство по тестированию (Testing Guide)

Качество и производительность `DDoS Detection System` обеспечивается модульными тестами и скриптами для бенчмаркинга.

## 1. Юнит-тесты (C++)

Мы используем кастомную легковесную систему тестирования (без подключения тяжеловесных библиотек типа GTest) для проверки основных компонентов: парсинга протоколов и загрузки конфигов.

Тесты расположены в папке `src/tests/` (или `tests/`).

### Как собрать и запустить тесты

В конфигурации CMake тесты собираются автоматически вместе с основным проектом.
Исполняемые файлы тестов находятся в папке `build/Release` (или вашей сборочной директории).

```cmd
# Перейдите в папку сборки
cd build/Release

# Запуск теста конфигурации
.\test_config.exe

# Запуск теста сериализации протокола
.\test_protocol.exe
```

Если все тесты пройдены успешно, программа вернет код возврата 0 и выведет `All tests passed!`.

## 2. Python скрипты для тестирования

В папке `scripts/` находятся инструменты для эмуляции нагрузки и верификации исправлений:

### 2.1 Генератор нагрузки (DDoS Emulator)
Скрипт `ddos_test.py` позволяет отправить UDP/TCP флуд на локальный интерфейс (Loopback) для проверки срабатывания коллектора и монитора.

```bash
# Установка зависимостей (если нужно)
pip install -r scripts/requirements.txt

# Запуск атаки на 10 секунд с размером пакета 1024 байта
python scripts/ddos_test.py --duration 10 --size 1024
```

### 2.2 Скрипты верификации (verify_fix.py)
Скрипты `verify_fix.py` и `verify_my_fix.py` используются во время разработки для быстрой проверки конкретных багов (например, парсинга JSON ответов от коллектора). 

Запуск:
```bash
python scripts/verify_my_fix.py
```
Они эмулируют TCP сервер коллектора и проверяют, как `ddos_monitor` реагирует на заданные пакеты данных.

## 3. Производительность (Benchmarks)

В проекте есть инструмент `run_benchmark.exe` (собирается из `tests/run_benchmark.cpp`).
Он тестирует пропускную способность `moodycamel::ConcurrentQueue` и скорость извлечения признаков без влияния сетевой задержки.

```cmd
# Запуск бенчмарка
.\run_benchmark.exe
```
Результат покажет максимальное количество пакетов в секунду (PPS), которое может "переварить" ваша конфигурация железа (обычно > 130,000 PPS на CPU уровня Core i5/Ryzen 5).
