# Руководство по тестированию (Testing Guide)

Качество и производительность `DDoS Detection System` обеспечивается модульными тестами и скриптами для бенчмаркинга. 

**Важное замечание**: После перехода на `libpqxx` (PostgreSQL), для успешного прохождения тестов базы данных необходимо наличие запущенного сервера PostgreSQL и корректно настроенного пароля.

## 1. Юнит-тесты (C++)

Мы используем систему модульного тестирования на базе `QtTest` для проверки основных компонентов: парсинга протоколов, загрузки конфигураций, работы с буфером файлов и инференса моделей.

### Как собрать и запустить тесты

В конфигурации CMake тесты собираются автоматически. Рекомендуется использовать MSVC и vcpkg для управления зависимостями (`libpqxx`, `PostgreSQL`).

```powershell
# Установка пароля для тестов БД (если отличается от дефолтного 'postgres')
$env:DDOS_DB_PASS="qwerty"

# Перейти в папку сборки
cd build/Release

# Запуск всех тестов
.\ddos_unit_tests.exe
```

## 2. Python скрипты для тестирования

В папке `scripts/` находятся инструменты для эмуляции нагрузки и верификации исправлений:

### 2.1 Проверка базы данных
Скрипт `check_db.py` позволяет быстро проверить подключение к PostgreSQL и наличие необходимых таблиц.
```bash
python scripts/check_db.py
```

### 2.2 Генератор нагрузки (DDoS Emulator)
Скрипт `ddos_test.py` позволяет отправить UDP/TCP флуд на локальный интерфейс (Loopback) для проверки срабатывания коллектора и монитора.

```bash
# Запуск атаки на 10 секунд с размером пакета 1024 байта
python scripts/ddos_test.py --duration 10 --size 1024
```

## 3. Производительность (Benchmarks)

В проекте есть инструмент `db_benchmark_qt.exe` (собирается из `tests/unit/benchmark_DatabaseManager.cpp`). Он замеряет скорость вставки данных в PostgreSQL с использованием различных техник `libpqxx` (row-by-row vs `stream_to` / COPY).

```cmd
# Запуск бенчмарка (требует DDOS_DB_PASS)
$env:DDOS_DB_PASS="qwerty"
.\db_benchmark_qt.exe
```

Результаты бенчмарков документируются в файле [PERFORMANCE_REPORT.md](PERFORMANCE_REPORT.md). Текущая реализация на `libpqxx` позволяет достигать скорости до 25,000 записей в секунду при использовании пакетной вставки через потоки (COPY).
