# Рекомендации по дальнейшему развитию

  

Дата актуализации: 08.04.2026

  

## 1. Приоритет P0 (критично)

  

### 1.1 Внешняя конфигурация

  

Сейчас часть параметров захардкожена или задается только CLI. Рекомендуется добавить единый config (JSON/YAML):

  

- tcp_port

- db_host/db_port/db_name/db_user

- model_path

- ep

- окно анализа

  

### 1.2 Надежная валидация модели

  

При загрузке ONNX-модели проверять:

  

- размер входного тензора;

- совместимость с количеством признаков;

- наличие и корректность scaler_params;

- допустимый диапазон confidence.

  

### 1.3 Наблюдаемость

  

Добавить диагностические метрики:

  

- очередь (size_approx);

- dropped packets per window;

- latency infer;

- время flush БД.

  

## 2. Приоритет P1 (важно)

  

### 2.1 Улучшение ML-контура

  

- Ввести pipeline оценки качества отдельно для RF и MLP.

- Добавить отчеты drift train/live.

- Стабилизировать MLP на данных с реальных интерфейсов.

  

### 2.2 Безопасность канала реального времени

  

- Ввести версию протокола в сообщениях.

- Добавить auth token/handshake.

- Включить ограничение размера входящих команд.

  

### 2.3 Улучшение работы с дампами pcap

  

- Режим пакетной обработки папки pcap.

- Экспорт summary-отчета (CSV/JSON) после replay.

  

## 3. Приоритет P2 (эволюционно)

  

- Linux-профиль сборки при сохранении MSVC как основной ветки.

- Плагинная загрузка нескольких моделей для A/B сравнения в реальном времени.

- Веб-панель поверх PostgreSQL для удаленного просмотра.

- Развитие расширенной архитектуры DashboardWidget: добавление пользовательских виджетов через JSON-конфигурации (динамический layout).

- Расширение функционала SysTray: добавление быстрых действий (пауза/старт) прямо из контекстного меню трея.

  

## 4. Рекомендованный порядок внедрения

  

1. Конфигурация + валидация модели.

2. Метрики наблюдаемости и алерты.

3. Улучшение ML-датасета и калибровки.

4. Расширение replay и отчетности.
