## Proposed Changes

### 1. Архитектура UI в стиле Grafana / Monium
Согласно вашим требованиям, интерфейс будет переработан для соответствия подходам систем наблюдаемости:
- **Боковое меню (Sidebar)**: Левая панель навигации (иконки/текст) для переключения между "Главным дашбордом", "Журналом событий (Logs)" и новой вкладкой "Аналитика (Explore/Metrics)".
- **Строки (Rows) и Скролл**: Главный дашборд будет обернут в `QScrollArea`. Панели будут сгруппированы в сворачиваемые/логические строки (Например: `Overview`, `Protocols & Ports`, `System Health`).
- **Time Picker**: Перенос управления масштабом времени в верхнюю навигационную панель дашборда.
- **Единое пространство**: Соблюдение отступов, скругленных углов панелей и цветовой палитры (Dark Mode / Light Mode).

#### [MODIFY] `src/monitor_ui/MainWindow.hpp/cpp` (или класс, где находится QTabWidget)

### 2. Этап 3.1: Продвинутые графики (Размещение в "Rows")
- **StackedAreaChart (Трафик по протоколам)**: Замена текущих линий на `QAreaSeries`. Данные TCP, UDP, ICMP будут суммироваться (стекаться) для создания единого "купола" трафика.
- **Heatmap (Тепловая карта)**: Кастомный виджет `HeatmapWidget` (на базе `QTableWidget` или `QPainter`), отображающий интенсивность запросов (Y — Топ-Порты, X — Время).
- **Гистограмма размеров пакетов**: `QBarSeries` в отдельной панели. Баскеты (64B, 128B, 512B, 1024B, >1024B) собираются на стороне коллектора.

#### [MODIFY] `src/network/FeatureExtractor.hpp/cpp`
#### [MODIFY] `src/common/Protocol.hpp/cpp`

### 3. Этапы 3.4 и 3.15: AlertGrid, SLO и Topology Map
- **AlertGridWidget**: Сетка "здоровья". Ячейки представляют подсети или сервисы (зеленый/красный в зависимости от `drop_rate`).
- **TopologyWidget (Traces/Flows)**: Наглядный граф прохождения трафика (`Internet -> Collector -> Target`). На основе стиля "Traces" из Monium, где можно увидеть "узкие горлышки". Будет размещен в строке `Network Topology` дашборда.

#### [NEW] `src/monitor_ui/AlertGridWidget.hpp/cpp`
#### [NEW] `src/monitor_ui/TopologyWidget.hpp/cpp`

### 4. Этап 3.13: Таймлайн инцидентов и Логи
- **Интерфейс логов**: Интеграция таблицы логов (`EventLogWidget`) с умным фильтром, похожим на *Metrics Explorer* в Monium.
- **Таймлайн**: Горизонтальная полоса (uptime bar), где красным цветом отмечаются периоды атак. Бесшовная связь: клик по красному квадратику таймлайна фильтрует логи по этому отрезку времени.

#### [NEW] `src/monitor_ui/TimelineWidget.hpp/cpp`

### 5. Этап 4.4: JSON-конфиг маппинга фичей
- Подключение `nlohmann/json` через `FetchContent` в `CMakeLists.txt`.
- Загрузка файла `<model_name>_features.json`. Порядок фичей в JSON будет диктовать порядок элементов в векторе, возвращаемом `getFeatures()`, чтобы исключить ошибки совместимости (когда Python обучил модель с одним порядком колонок, а C++ отправляет в другом).
