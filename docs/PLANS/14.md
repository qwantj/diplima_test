# Fix PPS Graph Bug
Причина проблемы заключается в том, что в `DashboardWidget` графики обновляются из двух разных источников:
1. `updateRealtime` — вызывается каждые 2 секунды (когда происходит инференс окна) и содержит полные данные (pps, tcp, udp, icmp, other).
2. `updateSnapshot` — вызывается каждую секунду (для обновления текстовой статистики) и в данный момент **добавляет нули** `0` для всех протоколов (tcp, udp, icmp, other), а также добавляет текущий `pps` со сбитым таймингом. Из-за этого на графике появляются резкие падения (импульсы) до нуля каждую секунду.

## Proposed Changes

### [Component Name] Dashboard UI
Изменение логики обновления графиков, чтобы избежать записи нулей.

#### [MODIFY] [DashboardWidget.cpp](file:///c:/Dev/CXX/diploma_test/src/monitor_ui/DashboardWidget.cpp)
- В методе `DashboardWidget::updateSnapshot` удалить весь код, который добавляет точки в ряды графиков (`m_ppsSeries->append`, `m_tcpLine->append` и т.д.).
- Оставить в `updateSnapshot` только обновление текстовых меток (`m_packetsLabel`, `m_ppsLabel`, `m_attackIndicator`).
- Всю отрисовку на графике оставим исключительно за `updateRealtime`, так как она предоставляет непрерывные и корректные данные с разбивкой по протоколам, что обеспечит ровные линии на графике.
