# Реализация финальных компонентов (GeoIP, Таймлайн)

Анализ кодовой базы показал, что бóльшая часть задач из вашего списка уже **добавлена в прошлых итерациях**, но, возможно, требует инспекции или финальной интеграции:
1. **3.1 (StackedAreaChart, Heatmap, Histogram)**: Уже реализовано в `DashboardWidget.cpp` (используются `QAreaSeries`, `HeatmapWidget` и гистограмма размеров пакетов).
2. **3.4 (AlertGrid)** и **3.15 (Topology Map)**: Реализованы внутри `DashboardWidget.cpp` и отображены на UI.
3. **4.4 (JSON-конфиг маппинга)**: В `FeatureExtractor::loadScalerParams` уже используется `nlohmann_json` для парсинга ключа `"features"`, и порядок фичей динамически применяется.

Ниже представлен план для **оставшихся нереализованными** пунктов.

## Proposed Changes

### Комплекс 3.3 — GeoIP локация источников через libmaxminddb
- **Зависимости**: Интеграция `libmaxminddb`. Будем использовать `FetchContent` в `CMakeLists.txt` или ожидать, что вы установите через `vcpkg install libmaxminddb`.
- **Новый класс `GeoIPResolver`** (`src/network/GeoIPResolver.hpp/cpp`):
  Обертка над C-API MaxMind (MMDB) для преобразования IPv4 в ISO-код страны и географические координаты. База `GeoLite2-City.mmdb` будет ожидаться в директории `models/`.
- **Geo-инфографика на Дашборде**: 
  Так как интеграция полноценной карты через `QWebEngine` сильно утяжелит проект, предлагается реализовать `GeoTableWidget` (распределение трафика по странам) + графическое представление (Светофор стран).

---

### Комплекс 3.13 — Таймлайн состояния и Гистограмма инцидентов
Ориентировано на `EventHistoryWidget` в мониторе.
- **Гистограмма плотности атак**: Добавление BarChart-диаграммы над таблицей `security_events`, отображающей количество зафиксированных атак по часам или дням.
- **Таймлайн состояния системы (Uptime/Alert History)**: 
  Создание кастомного виджета-полосы (как на GitHub или в Grafana), где зеленые полосы означают работу без аномалий, красные — DDoS-инциденты, с возможностью навести мышь и увидеть детали.

---
