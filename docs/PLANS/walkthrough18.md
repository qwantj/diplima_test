# Анализ и Доработка DDoS Detection: Итоги

В рамках данной сессии были реализованы оставшиеся критические и средние пункты плана разработки программного комплекса (DDoS Detection).

## Выполненные задачи

### 1. Горячая замена модели (Hot Swap) [Пункт 4.2]
- Добавлен потокобезопасный механизм блокировки `std::shared_mutex` в `DetectionEngine`. Ранее использовалась полная остановка захвата, теперь модель парсится "на лету" с эксклюзивной блокировкой записи только на период замены указателей.
- В интерфейсе `monitor_main.cpp` появилась кнопка "Загрузить модель" в панели инструментов. 
- Команда `load_model` отправляется через `DataBridge` с поддержкой автоматического поиска конфигурации `scaler_params.json` рядом с выбранной `.onnx` моделью. Успешная замена отображается в `QStatusBar`.

### 2. Инфографика системных ресурсов (CPU, RAM) [Пункт 3.5]
- Реализован класс `SystemMetricsCollector` для Windows: используются нативные API методы `GetSystemTimes` и `GlobalMemoryStatusEx` для периодического замера использования ОЗУ и Центрального процессора ОС.
- В `DetectionResult` добавлены поля `sysCpuUsage` и `sysRamUsage` (в %). Значения сериализуются в формате JSON протокола DataBridge на каждом рабочем цикле окна захвата пакетов.
- В `DashboardWidget.cpp` отрисованы два радиальных графика (Donut Charts), которые гармонично встроены в нижнюю панель под графиками полосы пропускания и распределения портов.

### 3. Графические материалы и Чертежи [Пункт 5.0]
В корне проекта сгенерирован файл [diagrams.md](file:///c:/Dev/CXX/diploma_test/diagrams.md), который содержит исходный код в формате **Mermaid** для генерации обязательных чертежей ВКР:
1. **Структурная схема** архитектуры системы (Collector - BD - Monitor).
2. **Блок-схема алгоритма** инференса и обработки пакетов.
3. **ER-диаграмма** таблиц базы данных PostgreSQL.
4. **Схема взаимодействия** (Sequence diagram) протокола обмена между клиентом и сервером.
