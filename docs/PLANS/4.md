# Debugging Attack Detection Plan

## Goal
Fix the issue where `scripts/ddos_test.py` attacks are classified as 'Benign'.

## Current Status
- Packet capture works (Benign logs seen).
- `ddos_test.py` sends high rate (67k PPS).
- Model output is consistently 0 (Benign).

## Hypotheses
1. **Scaler Mismatch**: The 100-packet window aggregation produces feature values that, when normalized, look "average" to the model.
    - *Check*: Log normalized values.
2. **Model Expectation**: The model might be trained on *flows* (full sessions), while we are feeding it *windowed slices* (100 packets).
    - If the model expects a "Flow Duration" of seconds, but we give it microseconds/milliseconds segments, the normalized value is heavily skewed.
3. **Label Flip**: Maybe 0 is Attack and 1 is Benign? (Unlikely, but possible).
4. **JSON Parsing**: The manual parser might be failing to load correct means/scales.

## Plan

### 1. Enhanced Logging
- [x] Add raw feature logging.
- [ ] Add **normalized** feature logging to `FeatureExtractor`. (Done in this step).
- [ ] Add probability/confidence logging in `ModelInferencer`.

### 2. Verify JSON Parser
- Inspect if `m_means` and `m_scales` are actually loaded correctly.

### 3. Model Logic Review
- Check `ModelInferencer.cpp` for threshold logic. Default is usually `argmax` or `> 0.5`.

## Action Items
1.  **Modify C++**: Add normalized feature logging.
2.  **Re-run**: Ask user to run attack again.
3.  **Analyze**:
    - If normalized values are huge (e.g., +10.0 or -10.0), scaling is working but model is resistant.
    - If normalized values are NaN or Inf, parser failed.
    - If normalized values are near 0, then the attack traffic *looks* like the mean of the training data.
