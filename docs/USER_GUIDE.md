## 1. Запуск системы

### Шаг 1: Запуск коллектора

Откройте **терминал от имени администратора** (требуется для Npcap):

```powershell
# Получить список доступных интерфейсов
.\ddos_collector.exe --list-interfaces

# Запустить захват с нужного интерфейса
.\ddos_collector.exe -i "WiFi"
# или
.\ddos_collector.exe -i "Ethernet"
```

При успешном запуске в консоли появятся:
```
[info] DetectionEngine: initialized with model 'rf_model.onnx', EP=cpu
[info] TcpServer: listening on 127.0.0.1:50050
[info] Successfully connected to PostgreSQL database: ddos_detection_db
[info] DetectionEngine: inference loop started (window=2s)
```

### Шаг 2: Запуск монитора

```powershell
.\ddos_monitor.exe
```

Монитор автоматически подключается к коллектору (127.0.0.1:50050) и базе данных согласно `config.json`. При успешном подключении к коллектору в тулбаре появится зелёный индикатор **Live**.

## 2. Интерфейс монитора

### 2.1 Боковая панель (Sidebar)

| Пункт меню | Назначение |
|---|---|
| **Dashboard** | Главная панель: PPS-графики, статус атаки, метрики системы |
| **Deep Analytics** | Топология сети, тепловая карта портов, SLO-мониторинг |
| **Event Log** | Журнал всех классифицированных окон трафика |
| **Security Incidents** | Хронология обнаруженных атак (агрегированные инциденты) |
| **Sessions History** | Список всех прошлых сессий работы системы |

### 2.2 Тулбар

| Элемент | Назначение |
|---|---|
| **Открыть PCAP** | Загрузить файл для анализа в offline-режиме |
| **⏹ Live** | Вернуться к анализу живого трафика после PCAP replay |
| **⏺ Запись / ⏹ Стоп** | Включить/выключить запись дампов трафика в `pcap_dumps/` |
| **Model** | Выбрать активную ML-модель (`rf_model.onnx` или `mlp_model.onnx`) |
| **DB: Connected / Disconnected** | Статус подключения к базе данных |
| **● Live** | Статус подключения к коллектору |
| **Theme** | Переключить тему оформления (Dark / Light) |

## 3. Dashboard — Главная панель

### Статус бар (верхняя строка)

| Индикатор | Описание |
|---|---|
| **Collector:** | Connected / Disconnected |
| **Total Packets** | Накопленное число пакетов за сессию |
| **PPS** | Текущая скорость в пакетах/секунду |
| **Drop Rate** | Процент потерянных пакетов (перегрузка очереди) |
| **Sources** | Число уникальных IP-источников |
| **Flows** | Число активных потоков в текущем окне |
| **Auto-Block** | Галочка включения/выключения FirewallManager |
| **Model** | Имя загруженной модели |
| **P(attack)** | Вероятность атаки (confidence) |
| **Статус** | **NORMAL** (зелёный) или **ATTACK** (красный) |

### PPS-График

Многослойный временно́й график с отображением:
- 📈 **Total PPS** — суммарный трафик
- 🔵 **TCP** — TCP-пакеты
- 🟠 **UDP** — UDP-пакеты
- 🟢 **ICMP** — ICMP-пакеты
- 🟣 **Other** — прочие протоколы
- 🔴 **Confidence** — уверенность модели в наличии атаки

**Управление графиком:**
- **Колесо мыши** — масштабирование по времени
- **Drag** — прокрутка (pan)
- **Кнопка Reset** — сброс зума до режима автопрокрутки
- **Цветные квадраты** (чекбоксы) — показать/скрыть слои

### Donut-диаграммы (правая колонка)

- **CPU%** — загрузка процессора (системные метрики)
- **RAM%** — использование памяти
- **Traffic** — соотношение атак и нормального трафика

## 4. Deep Analytics — Глубокая аналитика

Переключиться: **Sidebar → Deep Analytics**

| Виджет | Описание |
|---|---|
| **SLO Alert Grid** | Сетка 4×8 прямоугольников: зелёный = OK, красный = атака. Визуализирует аптайм системы. |
| **Network Topology** | Граф: центральная нода = текущий хост, узлы = источники трафика, толщина рёбер = интенсивность |
| **Top Sources** | Таблица топ-5 источников трафика с IP и числом пакетов |
| **Top Targets** | Таблица топ-5 целевых IP:port |
| **Top Ports** | Круговая диаграмма — распределение по портам назначения |
| **Bandwidth** | График пропускной способности в Мбит/с |
| **Port Heatmap** | Тепловая карта: ось X — время, ось Y — порты, цвет — интенсивность |
| **Packet Size** | Гистограмма распределения размеров пакетов (5 бинов) |

## 7. Sessions History — История сессий

Список сессий из PostgreSQL. Каждая запись содержит:
- Дата/время начала и окончания
- Интерфейс или путь к PCAP-файлу
- Использованная модель
- Статистика: всего пакетов / атак / норма

**Клик** на сессию загружает её данные в Dashboard и Event Log.

---

## 8. Анализ PCAP-файла

1. Нажать **Открыть PCAP** в тулбаре.
2. Выбрать файл `.pcap` или `.pcapng`.
3. Dashboard очищается, начинается анализ.
4. По завершении в системном трее появится уведомление "Pcap replay completed."
5. Для возврата к live-трафику нажать **⏹ Live**.

---

## 9. Запись PCAP-дампов

1. Нажать **⏺ Запись** (кнопка становится красной "⏹ Стоп").
2. Все пакеты записываются в `pcap_dumps/` с именем `session_YYYYMMDD_HHmmss.pcap`.
3. Окна с обнаруженной атакой записываются отдельно (commit с label=1).
4. Для остановки — нажать **⏹ Стоп**.

---

## 10. Смена ML-модели

В тулбаре из выпадающего списка **Model** выберите нужную модель.  
Коллектор переключает модель **без перезапуска** (hot swap):
- Инференс на следующем окне будет выполнен с новой моделью.
- Соответствующий скейлер (`rf_scaler_params.json` / `mlp_scaler_params.json`) переключается автоматически.

---

## 11. Активное противодействие (Auto-Block)

1. В Dashboard поставить галочку **Auto-Block**.
2. При обнаружении атаки (`label=1`, `confidence > 0.4`) коллектор блокирует TOP-1 источник через `netsh advfirewall`.
3. Заблокированные IP отображаются в поле `blockedIps` в сообщении `stats`.
4. При завершении атаки (переход к `label=0`) все блокировки снимаются.

> ⚠️ **Важно:** требуются права администратора для работы FirewallManager.

---

## 12. Системный трей

Монитор продолжает работу в фоне в системном трее:
- **Нормальный режим** — значок компьютера
- **Режим атаки** — красный значок с крестом
- **Двойной клик** — развернуть/свернуть окно
- **Правая кнопка** — меню: Expand / Pause / Exit

## 13. Настройка через командную строку

Коллектор поддерживает полное управление через CLI без конфигурационного файла:

```powershell
# Полный пример с явными параметрами:
.\ddos_collector.exe `
  -i "WiFi" `
  --model models\mlp_model.onnx `
  --ep cpu `
  --tcp-port 50050 `
  --db-host localhost `
  --db-port 5432 `
  --db-name ddos_detection_db `
  --db-user postgres `
  --db-password my_password `
  --pcap-dir pcap_dumps

# Анализ файла без базы данных:
.\ddos_collector.exe --pcap C:\attacks\sample.pcap --model models\rf_model.onnx
```

## 14. Экспорт данных

Для экспорта истории событий в CSV:
1. Перейдите в **Event Log**.
2. В меню файла выберите "Export CSV" (или через `ReportGenerator`).
3. Файл сохраняется с санитизацией данных (защита от CSV-инъекций через `CSVUtils`).
