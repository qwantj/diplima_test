## 2. Базовая архитектура

  

- ddos_core как общая библиотека.

- ddos_collector как программа сбора и анализа.

- ddos_monitor как программа мониторинга и управления.

- PostgreSQL как постоянное хранилище событий.

- TCP-канал Collector <-> Monitor для обмена в реальном времени.

## 4. Риски и меры

  

| Риск | Влияние | Мера |

|---|---|---|

| Domain shift для MLP | Ложные отрицания | Дотренировка на live-подобных данных |

| Несовпадение model/scaler | Некорректный infer | Жесткая валидация пары файлов |

| Перегрузка очередей | Потери пакетов | MAX_QUEUE_SIZE + мониторинг dropRate |

| Потеря БД-соединения | Потеря истории | Ретраи и буферизация в памяти |

  

## 5. Критерии готовности к защите

  

- Успешный live-детект UDP flood в контролируемом сценарии.

- Успешный replay pcap из GUI.

- Корректная запись сессии и событий в PostgreSQL.

- Наличие полного комплекта диаграмм и технической документации.
