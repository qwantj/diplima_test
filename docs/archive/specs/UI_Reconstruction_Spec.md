# **Спецификация пользовательского интерфейса (UI/UX): DDoS Dashboard**

Документ предназначен для восстановления визуальной части и компоновки элементов приложения "DDoS Dashboard" на основе скриншотов последней рабочей версии.

## **1\. Общие элементы дизайна (Global UI)**

* **Стиль:** Современный темный интерфейс (Dark Mode) с плоским дизайном (Flat Design).  
* **Цветовая палитра:**  
  * Фон приложения: Темно-серый/антрацитовый (\#2b2b2b или похожий).  
  * Фон панелей/виджетов: Чуть более светлый серый для выделения на фоне (\#323232).  
  * Акценты (Нормальное состояние): Зеленый (текст статусов, графики здоровья, бейджи).  
  * Акценты (Предупреждения/Атаки): Красный (счетчик атак, графики) и Оранжевый/Желтый (вероятность, confidence).  
  * Текст: Светло-серый/белый.  
* **Структура окна:**  
  * **Верхняя панель (Header):** Зеленая полоса с названием приложения "DDoS Dashboard" и стандартными кнопками управления окном (справа).  
  * **Панель инструментов (Toolbar):** Под хедером. Содержит глобальные элементы управления.  
  * **Левая панель навигации (Sidebar):** Меню переключения между основными разделами приложения.  
  * **Рабочая область (Main Content):** Занимает оставшуюся часть экрана справа от Sidebar.

## **2\. Верхняя панель управления (Toolbar)**

Расположена горизонтально под заголовком окна.

* **Кнопка "Открыть PCAP":** Иконка папки (желтая) \+ текст.  
* **Кнопка выбора папки (только иконка):** Желтая папка.  
* **Выпадающий список "Модель:":** Выбор используемой модели ИИ (на скрине выбрана mlp\_model.onnx).  
* **Индикатор статуса:** Зеленый кружок с текстом "Live" (указывает на режим реального времени).  
* *(Справа)* **Выпадающий список "Theme:":** Выбор темы (на скрине Dark).

## **3\. Боковая панель навигации (Sidebar)**

Расположена слева, имеет темно-синий/серый оттенок. Активный пункт выделяется более светлым фоном.

Содержит следующие разделы (с соответствующими иконками):

1. **Dashboard** (Иконка: цветные блоки/график)  
2. **Deep Analytics** (Иконка: график с линией тренда)  
3. **Event Log (Live)** (Иконка: оранжевый документ/лог)  
4. **Security Incidents** (Иконка: синий щит)  
5. **Sessions History** (Иконка: фиолетовый список/терминал)

## **4\. Раздел "Dashboard" (Главный экран)**

Окно разделено на три горизонтальные зоны: строка статуса, главный график, нижние виджеты ресурсов.

### **4.1. Overview (Global State) \- Строка статуса**

Тонкая горизонтальная панель над главным графиком. Содержит метрики в один ряд:

* **Collector:** Статус соединения (например, Connected (Live) зеленым цветом).  
* **Total Packets:** Числовое значение (белый текст).  
* **Current PPS:** Числовое значение скорости.  
* **Drop Rate:** Значение pps (на скрине 0.0 pps красным цветом).  
* **Sources:** Число активных источников.  
* **Чекбокс "Auto-Block Top-IP (BPF)":** Включение автоматической блокировки.  
* **Иконка ИИ \+ Имя модели:** (напр. mlp\_model.onnx).  
* **Probability:** Процентное значение (на скрине 0.0% зеленым).  
* **Статус трафика:** Текст со значком галочки (напр., ✔ Normal Traffic зеленым цветом).

### **4.2. Главный график (Time-series Area Chart)**

Занимает центральную, самую большую часть экрана.

* **Тип:** График с областями (Area chart) и линиями.  
* **Оси:**  
  * **Ось X:** Время (Time) в формате ЧЧ:ММ:СС.  
  * **Ось Y (Левая):** PPS (Packets per Second).  
  * **Ось Y (Правая):** Confidence (Уверенность в атаке) от 0% до 100%.  
* **Легенда (вверху по центру):** Чекбоксы для включения/выключения слоев:  
  * Packets per Second (Синий)  
  * Other (Серый контур)  
  * ICMP (Красный контур)  
  * UDP (Оранжевый контур)  
  * TCP (Зеленый контур)  
  * Attack Confidence % (Оранжевая заливка области)  
* **Визуализация:** Пики графика показывают всплески трафика, заливка оранжевым показывает уровень уверенности модели в том, что это атака.

### **4.3. Нижняя панель виджетов (System & Traffic State)**

Три одинаковых квадратных панели, расположенных в ряд внизу. Каждая содержит кольцевой график (Donut chart).

1. **CPU (x.x%):**  
   * Кольцевой график показывающий загрузку ЦП.  
   * Легенда внизу: In Use (Красный) / Free (Зеленый).  
   * Процент загрузки указан в заголовке виджета.  
2. **RAM (x.x%):**  
   * Кольцевой график загрузки ОЗУ.  
   * Легенда: In Use (Оранжевый) / Free (Зеленый).  
3. **Traffic Ratio (Attack: x.x%):**  
   * Кольцевой график соотношения нормального и вредоносного трафика.  
   * Легенда: Normal (Зеленый) / Attack (Красный).

## **5\. Раздел "Deep Analytics"**

Сложный дашборд, разбитый на сетку (Grid) из различных виджетов для глубокого анализа сетевых пакетов.

### **Верхний ряд:**

* **Global SLO Health (Last points):** Матрица (сетка) из прямоугольных индикаторов (светодиодов). Зеленые прямоугольники (4 ряда по 8 столбцов на скрине) показывают здоровье инфраструктуры в последние моменты времени.  
* **Network Topology:** Граф (Node-Link). Слева синий узел Monitor, от которого идут связи к IP-адресам назначения (целям) справа. Над связями подписано количество пакетов (pkts). Сверху указан общий Ingress Traffic (Mbps).

### **Средний ряд (Таблицы и Порты):**

* **Таблица источников:** Столбцы \# (номер), IP Addr (IP источника), Packets (количество пакетов).  
* **Таблица целей:** Столбцы \#, Target (IP:Port) (IP назначения и порт), Packets.  
* **Top Active Ports:** Кольцевой график (Donut), показывающий распределение пакетов по портам назначения. Под графиком легенда: цветные квадратики с номерами портов (например, 443, 56389, 12041).

### **Нижний ряд:**

* **Network Bandwidth:** Линейный график (Line chart). Ось Y \- Mbps, Ось X \- Время. Показывает пропускную способность сети.  
* **Port Activity Heatmap:** Тепловая карта (Scatter plot). Ось Y (вероятно, порты), Ось X \- Время. Каждая точка представляет активность на определенном порту.  
* **Packet Size Distribution:** Столбчатая диаграмма (Bar chart). Ось X \- корзины размеров пакетов (0-64B, 65-128B, 129-512B, 513-1024B, \>1024B). Ось Y \- количество/частота.

## **6\. Раздел "Event Log (Live)"**

Журнал логов в реальном времени, представленный в виде большой таблицы на весь экран.

* **Верхняя панель (Фильтр):** Выпадающий список "All Events" (вероятно, позволяет фильтровать Benign/Malicious).  
* **Таблица данных:**  
  * Time: Время события (с миллисекундами, напр. 21:43:52.484).  
  * Flow: Идентификатор или описание потока (на скрине пусто).  
  * Label: Классификация трафика (Benign \- нормальный, зеленым цветом; или Attack красным).  
  * Confidence: Уверенность модели (от 0.00 до 1.00).  
  * PPS: Скорость (пакетов в секунду).  
  * Model: Путь к файлу модели, которая произвела классификацию (напр., models/rf\_model.onnx или полные пути).

## **7\. Раздел "Security Incidents"**

Дашборд для исторического анализа и реагирования на инциденты (атаки).

* **Панель управления:**  
  * Кнопки: Обновить, Экспорт в CSV.  
  * Фильтры (справа): Дата (с календарем), Тип (выпадающий список "Все типы"), IP (текстовое поле "Фильтр по IP...").  
* **Uptime / Временная шкала состояния (выбранный день):** Горизонтальная полоса (таймлайн), разбитая на квадраты (по часам/минутам). Отметки времени: 00:00, 06:00, 12:00, 18:00. Квадраты меняют цвет в зависимости от наличия атак в этот период (на скрине один квадрат светло-зеленый, остальные темно-серые).  
* **Таблица инцидентов:** Список зафиксированных атак за выбранный период.  
  * Столбцы: Время начала, Длительность (с), IP атакующего, Max PPS, Тип атаки, Уверенность.

## **8\. Раздел "Sessions History"**

История всех запущенных сессий захвата трафика (PCAP файлов или прослушивания интерфейсов). Представлена в виде простой таблицы.

* **Таблица сессий:**  
  * ID: Уникальный номер сессии (сортировка по убыванию).  
  * Start Time: Дата и время начала сессии (YYYY-MM-DD HH:MM:SS).  
  * Interface/PCAP: Имя сетевого интерфейса (напр., Wi-Fi, \\Device\\NPF\_Loopback) или абсолютный путь к анализируемому файлу .pcap.  
  * Model: Имя использованной модели ИИ.  
  * Attacks: Количество зафиксированных пакетов/атак. **Важно:** Если значение \> 0, оно выделяется **красным** цветом для привлечения внимания. Если 0 — белым.  
  * Total Packets: Общее количество обработанных пакетов в сессии.
