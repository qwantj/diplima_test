# Проблема детекции MLP-модели DDoS

  

Дата актуализации: 08.04.2026

  

## 1. Проблема

  

На реальном интерфейсе (например, Realtek Wi-Fi) MLP-модель может хуже определять атаки, чем RF, при одинаковом тесте через ddos_test.py.

  

## 2. Что уже подтверждено

  

- RF и MLP обе загружаются в ONNX Runtime.

- Логика выбора scaler строится по имени модели и fallback на scaler_params.json.

- FeatureExtractor формирует 8 признаков окна.

- Окно инференса составляет 1 секунда.

  

## 3. Наиболее вероятные причины

  

1. Domain shift.

  

Реальные распределения признаков (особенно PPS и длины пакетов) отличаются от train-выборки.

RF обычно устойчивее к таким смещениям, чем MLP.

  

2. Saturation после нормализации.

  

При очень больших значениях признаков MLP может попадать в область, где выходная вероятность калибрована плохо.

  

3. Недостаточная репрезентативность train-данных.

  

Если MLP обучалась преимущественно на лабораторных данных, профиль realtek/live может быть представлен слабо.

  

4. Неполная согласованность model/scaler.

  

Даже незначительная ошибка в порядке признаков или в scaler приводит к деградации нейросети.

  

## 4. Диагностический чеклист

  

1. Проверить, что для MLP используется корректный файл *_scaler_params.json.

2. Логировать raw и normalized признаки в live-режиме (выборочно).

3. Сравнить train/live распределения по каждому признаку.

4. Проверить confidence histogram для RF и MLP.

5. Проверить threshold принятия решения для MLP.

  

## 5. Практические рекомендации

  

- Дообучить MLP на live-подобных данных с реального интерфейса.

- Добавить калибровку (Platt/Isotonic) на валидационном наборе.

- Рассмотреть clipping экстремальных значений или use_log1p.

- Для защиты ВКР использовать RF как production baseline, а MLP — как экспериментальную альтернативу с указанием ограничений.

  

## 6. Что показывать на защите

  

- Сравнение RF и MLP на одинаковых replay/live сценариях.

- Причины различий и план улучшения MLP.

- Обоснование выбора рабочей модели для текущей версии комплекса.
