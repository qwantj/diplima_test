# Индивидуальная стратегия предобработки для моделей

Цель: RF работает с оригинальными признаками (через `StandardScaler`), а MLP — с предварительно прологарифмированными (через `Log1p` -> `StandardScaler`).

Решение: добавляем флаг `"use_log1p"` прямо в файлы `*_scaler_params.json`. C++ движок и `evaluate.py` будут читать этот флаг и понимать, что нужно делать перед применением скейлера.

## Proposed Changes

### Python

#### [MODIFY] [train.py](file:///c:/Dev/CXX/diploma_test/scripts/train.py) & [colab_train_code.py](file:///c:/Dev/CXX/diploma_test/scripts/colab_train_code.py)
Разделяем обучение:
- RF: `X_train_rf = rf_scaler.fit_transform(X_train)`
- MLP: `X_train_mlp = mlp_scaler.fit_transform(np.log1p(X_train))`

Добавляем параметр `use_log1p` в `save_scaler_params`:
```python
def save_scaler_params(scaler, features, path, use_log1p=False):
    params = {
        "features": features,
        "mean": scaler.mean_.tolist(),
        "scale": scaler.scale_.tolist(),
        "use_log1p": use_log1p
    }
```
Сохранение:
```python
save_scaler_params(rf_scaler, FEATURES, "rf_scaler_params.json", use_log1p=False)
save_scaler_params(mlp_scaler, FEATURES, "mlp_scaler_params.json", use_log1p=True)
```
