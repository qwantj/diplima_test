# Интеграция произвольных ONNX-моделей и динамических признаков

В рамках этой задачи была проведена фундаментальная модернизация движка детектирования. Теперь DDoS-детектор больше не ограничен жестким списком из 8 фичей. Аналитика собирает необходимые данные динамически, что позволяет легко подключать новые (более сложные) ONNX-модели с любым количеством параметров без перекомпиляции ядра.

## Достигнутые результаты

### 1. Архитектура параметров (`FeatureExtractor` и `ModelInferencer`)

- **Динамический сбор признаков**: Массив фичей для ONNX-модели теперь формируется на основе конфигурации (`scaler_params.json`). Модель получает ровно то количество параметров, которое она ожидает.
- **Поддержка Мультиклассификации**: В `ModelInferencer` добавлено извлечение как логических меток класса, так и вероятностей предсказания атаки независимо от фреймворка, в котором обучали модель с нейросетью или Scikit-Learn классификатор.
- **Маппинг названий**: Добавлено извлечение названий классов атак (например: `Normal`, `SYN Flood`, `UDP Flood`) из JSON-конфигурации в `FeatureExtractor`.

### 2. СУБД и Хранение событий (`DatabaseManager` & `init.sql`)

- **JSONB сериализация**: Жестко заданные аналитические колонки (типа `flow_duration`, `fwd_packets` и т.д.) заменены на единую колонку `features_json` для обеспечения гибкости хранения любых параметров любых будущих моделей.
- **Новая колонка `attack_type`**: Добавлена в базу данных для хранения точного имени типа атаки, определенного моделью.
- **Обновлен скрипт `init.sql`**: Подготовлен скрипт для миграции существующих логов (С использованием операторов `ALTER TABLE ... DROP COLUMN`).

### 3. Пользовательский интерфейс (`EventHistoryWidget`)

- Интерфейс `EventHistory` теперь динамически заполняет выпадающий список (Комбо-бокс) для фильтрации типов атак (`m_typeFilter`). При загрузке новых моделей и поступлении свежих событий, новые классы атак автоматически становятся доступны для фильтрации в GUI.
- Все связанные с интерфейсом классы были перенастроены для чтения и записи строкового поля `attack_type`.

> [!TIP]
> При необходимости использования старой базы данных без потери записей, необходимо зайти в ваш сервер PostgreSQL и запустить SQL-запрос `ALTER TABLE` из нового `sql/init.sql` из корня проекта.

## Тестирование
Код успешно скомпилирован компилятором MSVC + Qt 6 под архитектуру x64 и готов к запуску в production. Связка параметров между ML-моделью, сборщиком пакетов и базой данных теперь полностью синхронизирована.
