# Улучшение графиков PPS: автоподстройка Y-оси и зум

## Описание проблем

1. **Y-ось не адаптируется к видимому диапазону** — `adjustAxes()` ищет максимум по **всем** точкам серии, а `m_maxPps` только увеличивается (никогда не уменьшается). При скролле по X-оси график может выглядеть слишком маленьким или слишком большим.

2. **Зум колёсиком работает некорректно** — `chart()->zoom(factor)` зумит обе оси одновременно и от центра чарта, а не от позиции курсора. Пользователь ожидает зум по оси X с привязкой к позиции курсора.

**1. `adjustAxes()` — автоподстройка Y к видимому диапазону:**

Вместо поиска максимума по всем точкам, находим min/max Y только среди точек, попадающих в текущий видимый диапазон X-оси. Добавляем 10% отступ сверху. Минимум Y ставим в 0 (PPS не может быть отрицательным). Если видимых точек нет — оставляем Y как есть.

```cpp
void DashboardWidget::adjustAxes() {
    if (m_ppsSeries->count() == 0) return;

    const auto& pts = m_ppsSeries->points();
    qint64 firstX = static_cast<qint64>(pts.first().x());
    qint64 lastX  = static_cast<qint64>(pts.last().x());

    if (m_startTime == 0) m_startTime = firstX;

    if (m_autoScroll) {
        qint64 winStart = lastX - VISIBLE_WINDOW_MS;
        if (winStart < m_startTime) winStart = m_startTime;
        m_axisX->setRange(QDateTime::fromMSecsSinceEpoch(winStart),
                          QDateTime::fromMSecsSinceEpoch(lastX));
    }

    // Определяем видимый диапазон X
    qint64 visMinX = m_axisX->min().toMSecsSinceEpoch();
    qint64 visMaxX = m_axisX->max().toMSecsSinceEpoch();

    // Находим min/max Y только для видимых точек
    float visMaxY = 0.0f;
    bool found = false;
    for (const auto& p : pts) {
        qint64 px = static_cast<qint64>(p.x());
        if (px >= visMinX && px <= visMaxX) {
            float py = static_cast<float>(p.y());
            if (!found || py > visMaxY) visMaxY = py;
            found = true;
        }
    }

    if (found) {
        float upperY = std::max(visMaxY * 1.15f, 10.0f); // минимум 10 чтобы ось не схлопывалась
        m_axisY->setRange(0, upperY);
    }
}
```

**2. `wheelEvent()` — зум вокруг позиции курсора, только по оси X:**

Вместо `chart()->zoom()`, вручную пересчитываем диапазон X-оси вокруг точки под курсором. Y-ось не трогаем — она подстраивается автоматически через `adjustAxes()`.

```cpp
void InteractiveChartView::wheelEvent(QWheelEvent* event) {
    if (!chart() || chart()->series().isEmpty()) { event->accept(); return; }
    
    const qreal factor = (event->angleDelta().y() > 0) ? 0.8 : 1.25;
    
    // Получаем оси
    auto axes = chart()->axes(Qt::Horizontal);
    if (axes.isEmpty()) { event->accept(); return; }
    auto* axisX = qobject_cast<QDateTimeAxis*>(axes.first());
    if (!axisX) { event->accept(); return; }
    
    // Позиция курсора в координатах графика
    QPointF chartPos = chart()->mapToValue(chart()->mapFromScene(
                           mapToScene(event->position().toPoint())));
    
    qint64 minX = axisX->min().toMSecsSinceEpoch();
    qint64 maxX = axisX->max().toMSecsSinceEpoch();
    qint64 centerX = static_cast<qint64>(chartPos.x());
    
    // Зумим вокруг позиции курсора
    qint64 newMin = centerX - static_cast<qint64>((centerX - minX) * factor);
    qint64 newMax = centerX + static_cast<qint64>((maxX - centerX) * factor);
    
    // Минимальный период — 5 секунд
    if (newMax - newMin < 5000) { event->accept(); return; }
    
    axisX->setRange(QDateTime::fromMSecsSinceEpoch(newMin),
                    QDateTime::fromMSecsSinceEpoch(newMax));
    
    emit userInteracted();
    event->accept();
}
```

**3. Вызовы `adjustAxes()` после зума/скролла:**

После `wheelEvent` и `mouseMoveEvent` (при пан), нужно вызывать `adjustAxes()` для пересчёта Y. Для этого в `InteractiveChartView` добавим сигнал `viewChanged()` или вызовем `adjustAxes()` из `DashboardWidget` по сигналу `userInteracted`.

Для минимальности изменений — подключим `adjustAxes()` к `userInteracted` сигналу.

**4. Остальные изменения:**

- В `loadHistory()` — убрать ручное выставление `m_maxPps`, т.к. `adjustAxes()` теперь справляется сама.
- Удалить ставшее ненужным поле `m_maxPps`.
