# Сохранение PCAP дампов с маркировкой (v2)

Потоковая запись в PCAP с маркировкой benign/attack, без буферизации в RAM.

## Архитектура

```
handlePacket() ──→ PcapDumper::writePacket(rawPacket)
                         │  (пишет в _temp.pcap потоково, 0 RAM)
                         │
   inference (каждые 2с) ──→ PcapDumper::finalizeWindow(label)
                                 │
                                 ├─ close _temp.pcap
                                 ├─ читает temp, дописывает в benign.pcap или attack.pcap (append)
                                 └─ удаляет _temp.pcap
```

**Результат:** 2 файла на сессию — `session_DIR/benign.pcap` и `session_DIR/attack.pcap`.

## Решение рисков

| Риск | Решение |
|------|---------|
| OOM (буферизация в RAM) | Потоково пишем в temp-файл через `PcapFileWriterDevice` |
| Disk I/O блокирует захват | Файловые операции в отдельном потоке через очередь задач |
| Файловый спам | 2 файла на сессию (benign.pcap + attack.pcap), temp удаляется |
| Deep copy RawPacket | `PcapFileWriterDevice::writePacket()` копирует данные внутри |

- `startSession(outputDir)` — создаёт `pcap_dumps/session_YYYYMMDD_HHmmss/`, открывает `benign.pcap` и `attack.pcap`
- `writePacket(RawPacket)` — пишет пакет в текущий temp-файл (потоково, API PcapPlusPlus)
- `finalizeWindow(label)` — закрывает temp, дописывает его содержимое в соответствующий сессионный файл
- `stopSession()` — финализирует последнее окно, закрывает файлы
- I/O в отдельном потоке через `std::queue` + `condition_variable`
