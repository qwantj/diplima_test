# Добавление выбора модели для предсказания DDoS атак

Сейчас приложение жёстко привязано к `rf_model.onnx`. В каталоге `models/` также есть `mlp_model.onnx`. Необходимо дать пользователю возможность выбрать модель из списка доступных прямо в UI — до или во время работы.

## Proposed Changes

### DetectionEngine (backend)

#### [MODIFY] [DetectionEngine.hpp](file:///c:/Dev/CXX/diploma_test/src/core/DetectionEngine.hpp)

- Добавить `bool switchModel(const std::string& modelPath)` — метод для смены модели «на лету» (перезагружает ONNX-сессию в `ModelInferencer`).
- Хранить `m_currentModelPath` для отслеживания текущей модели.

#### [MODIFY] [DetectionEngine.cpp](file:///c:/Dev/CXX/diploma_test/src/core/DetectionEngine.cpp)

- Реализовать `switchModel()`: вызывает `m_inferencer->loadModel(newPath)` и обновляет `m_currentModelPath`. Возвращает `false` при ошибке.

---

### UI (main.cpp)

#### [MODIFY] [main.cpp](file:///c:/Dev/CXX/diploma_test/src/main.cpp)

- Добавить `QComboBox* m_modelCombo` — выпадающий список моделей.
- В `setupUI()` добавить секцию «Model» внутри Control Panel (или рядом) с лейблом и комбобоксом.
- В `setupEngine()`:
  - Сканировать каталог `models/` через `QDir` и найти все `*.onnx` файлы.
  - Заполнить `m_modelCombo` найденными файлами (отображаемое имя — без расширения, data — относительный путь `models/filename.onnx`).
  - Инициализировать движок первой моделью из списка (по умолчанию — `rf_model.onnx`, если он есть).
- Подключить сигнал `currentIndexChanged` к слоту `onModelChanged()`.
- `onModelChanged()`:
  - Блокировать смену, если идёт захват (показать предупреждение).
  - Вызвать `m_engine.switchModel(newPath)`.
  - Логировать смену модели в `m_logText`.
