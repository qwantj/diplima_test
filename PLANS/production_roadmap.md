# План реализации оставшихся функций (Production Roadmap)

Дата создания: 10.05.2026
Статус: В работе

## Фаза 1: Исправление критических ошибок связи и управления (Collector Command Handling)
- [x] **TcpServer**: Исправить механизм передачи команд. Заменить ошибочный вызов `newConnection()` на специализированный сигнал `commandReceived(std::string cmd, nlohmann::json data)`.
- [x] **Collector Main**: Реализовать цикл обработки команд в `collector_main.cpp`. 
    - Поддержка `load_pcap` (динамическое переключение на анализ файла).
    - Поддержка `load_model` (горячая замена ONNX модели).
    - Поддержка `config_bpf` (удаленная настройка фильтров захвата).
- [x] **DetectionEngine**: Добавить явный статус завершения replay для корректной отправки уведомления `replay_done`.

## Фаза 2: Агрегация инцидентов безопасности (Incident Detection & Logging)
- [x] **DetectionEngine**: Реализовать State Machine для детекции атак (Normal -> Under Attack -> Normal).
- [x] **Аналитика**: Сбор агрегированных данных за период атаки (Start Time, Duration, Max PPS, Top Attacker IP).
- [x] **DatabaseManager**: Реализовать метод `enqueueSecurityEvent` для записи инцидентов в таблицу `security_events`.
- [x] **Интеграция**: Автоматическое сохранение инцидента в БД при окончании атаки.

## Фаза 3: Расширение функционала SysTray и UX
- [x] **SysTray**: Добавить контекстное меню (Развернуть, Пауза, Выход).
- [x] **Визуализация**: Динамическая смена иконки трея в зависимости от уровня угрозы (Зеленый/Красный).
- [x] **Dashboard**: Реализовать отображение метрики `Drop Rate` (потери пакетов при перегрузке очереди).
- [x] **Analytics**: Добавить экспорт отчетов об инцидентах в CSV из `SecurityIncidentsWidget`.


## Фаза 4: Конфигурация и Наблюдаемость
- [x] **ConfigManager**: Вынос хардкода (порты, реквизиты БД, пути к моделям) в `config.json`.
- [x] **Протокол**: Внедрение версионирования в JSON-сообщения (`"version": "2.2"`).
- [x] **Health Check**: Добавить в телеметрию данные о состоянии внутренних очередей и времени инференса (latency).
